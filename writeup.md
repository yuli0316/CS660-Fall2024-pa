## Describe any design decisions you made.
Dbfile:
I divided the file into fixed size pages (PAGE_SIZE). The number of pages is calculated by dividing the file size by the page size (st_size/PAGE_SIZE). This design decision simplifies page management because it relies on a fixed size structure for each page.
heapage&tuple: Tuples are stored in the page as serialized sequences of bytes. I use the insertTuple() method to calculate the correct byte offset of the tuple data based on the slot number, and then serialize the tuple to that location (serialized and deserialized are defined in tuple.cpp).
When requested, the getTuple() method deserializes the byte data back into the Tuple object. This allows for flexible storage of tuple data, and TupleDesc ensures that tuples are serialized and deserialized according to the correct format.
